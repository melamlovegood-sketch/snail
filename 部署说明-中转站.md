# 部署说明 · 新增「自定义 / 中转站」AI 供应商

这个分支给 AI 配置加了一个**自定义 / 中转站**选项，用户可以填中转站的 baseURL + API Key + 模型名（OpenAI 兼容格式）来调用。同时把「截图解析任务」原本写死的 `qwen-vl-plus` 改成了**可配置的视觉模型**。

## 改了哪些文件

- `snail-api/src/index.js` —— **后端 Worker**：`handleQwen` 新增 `provider === 'custom'` 分支，用 `normalizeCustomEndpoint(baseURL)` 把地址规整成 `/v1/chat/completions` 再转发。
- `index.html` —— **前端**：设置页新增「自定义 / 中转站」选项 + baseURL 输入框 + 独立的「视觉模型」字段；`aiConfig` 新增 `baseURL`、`visionModel` 两个字段（向后兼容旧配置）。
- `sw.js` —— 仅 `CACHE_VERSION` 时间戳更新（强制客户端刷新缓存）。

## 部署步骤（顺序很重要：先 Worker，再前端）

> ⚠️ **一定先部署 Worker 再部署前端**。新前端的请求体里多了一个 `baseURL` 字段，新 Worker 会把它正确剥离；如果前端先上线而 Worker 还是旧的，旧 Worker 会把这个字段透传给上游接口，可能导致调用异常。

### 第 1 步：部署 Worker

```bash
git checkout feat/custom-relay-provider   # 切到本分支（或合并后在 main 上）
cd snail-api
npx wrangler deploy
```

部署完这一步，中转站功能的后端就生效了。

### 第 2 步：发布前端

把根目录的 `index.html` 重新发布到 Cloudflare Pages（按你原来的发布方式即可）。发布前记得已经跑过 `npm run bump`（本分支已经跑过，`sw.js` 版本号已更新）。

## 怎么验证

1. 打开 app → 设置 → AI 模型配置，供应商选「自定义 / 中转站」。
2. 填入中转站的 **接口地址（baseURL）**（如 `https://你的中转站.com/v1`）、**API Key**、**模型（文字）**（如 `gpt-4o`）。
3. 如果要用截图解析，**视觉模型**填一个支持多模态识图的模型（如 `gpt-4o`）。
4. 在助手里用文字描述生成任务 → 应成功；上传截图 → 应能提取任务。
5. 回归检查：把供应商切回千问，文字 + 图片解析仍应正常。

## 一个需要知道的行为变化

图片解析以前**无论选哪个供应商都强制走千问 `qwen-vl-plus`**（并回退到服务端千问 Key）。现在改为**走用户配置的视觉模型**。

- 千问用户：默认视觉模型就是 `qwen-vl-plus`，**行为不变**。
- 非千问用户：需要在设置页把「视觉模型」填成对应供应商的多模态模型（如 OpenAI 填 `gpt-4o`），否则图片解析会因模型名不匹配而失败。

这是本次需求有意为之的设计（让图片解析也能用中转站 / 其它供应商）。
